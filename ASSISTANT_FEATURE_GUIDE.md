# 🤖 JellyStudy 智能助手功能说明

## 📋 功能概述

智能助手是一个悬浮窗形式的AI交互组件，为JellyStudy平台提供两大核心功能：

### 1️⃣ **AI对话平台** - 直接与DeepSeek大模型交流
- 💬 多轮对话支持（保留上下文）
- 🎓 学术和技术问题解答
- 💡 学习建议和概念解释
- ⚡ 实时响应（约5-15秒）

### 2️⃣ **知识图谱生成器** - 可视化知识点关联
- 📌 标记感兴趣的知识点
- 🔗 AI分析知识点间的关联关系
- 🗺️ 交互式知识图谱可视化
- 🎨 力导向图布局，支持拖拽

## 🚀 快速开始

### 启动方式

1. **重新构建并启动Docker容器**
```bash
cd c:\onlywork\jellystudy
docker compose up -d --build
```

2. **访问Web前端**
   - 打开浏览器访问：http://localhost:8080
   - 页面右下角会出现一个紫色的🤖机器人图标

3. **点击图标打开智能助手面板**

---

## 🎯 功能一：AI对话使用指南

### 界面位置
- 点击右下角的🤖图标
- 默认显示"💬 对话"标签页

### 使用步骤
1. 在输入框中输入你的问题
2. 按 **Enter键** 或点击"发送"按钮
3. 等待AI响应（约5-15秒）
4. 继续对话或提出新问题

### 示例对话
```
用户: 什么是微服务架构？
博识尊: 微服务架构是一种软件开发方法...

用户: 那它和单体架构有什么区别？
博识尊: 主要区别包括：1. 部署方式...（会记住上下文）
```

### 特色功能
- ✅ **上下文记忆**：自动保留最近10轮对话历史
- ✅ **Markdown支持**：AI回答支持列表、加粗等格式
- ✅ **错误处理**：网络异常时给出友好提示
- ✅ **打字体验**：Enter发送，Shift+Enter换行

---

## 🎯 功能二：知识图谱使用指南

### 步骤1：标记感兴趣的知识点

1. 访问任意**问题详情页** (http://localhost:8080/question/{id})
2. 找到知识点标签旁边的 **"📌 标记感兴趣"** 按钮
3. 点击按钮，知识点会被添加到兴趣列表
4. 右下角🤖图标的徽章数字会增加

**示例位置：**
```
问题标题下方：
[Java基础] [📌 标记感兴趣] 👤作者 📅时间
```

### 步骤2：查看和管理兴趣列表

1. 点击右下角🤖图标打开助手
2. 切换到 **"📚 知识图谱"** 标签页
3. 查看已标记的知识点列表
4. 可以单独移除某个知识点（点击✕按钮）
5. 或清空整个列表

### 步骤3：生成知识图谱

1. 在兴趣列表中确保有至少1个知识点
2. 点击 **"🗺️ 生成图谱"** 按钮
3. 等待AI分析关联关系（约10-20秒）
4. 查看生成的可视化图谱

### 图谱功能
- 🔄 **力导向布局**：节点自动排列
- 🖱️ **拖拽交互**：可拖动节点调整位置
- 🔍 **缩放平移**：鼠标滚轮缩放，拖拽空白处平移
- 💡 **悬停提示**：鼠标悬停显示详细信息
- 🎨 **颜色编码**：不同分类用不同颜色标识
- ➡️ **关系类型**：
  - 🔴 depends-on（依赖）
  - 🟢 contains（包含）
  - 🔵 related-to（相关）
  - 🟠 prerequisite-of（前置）

```
┌─────────────────────────────────────────┐
│           前端层 (Thymeleaf)            │
│  ┌──────────────┐  ┌─────────────────┐ │

│                 ▼                      │
│  ┌──────────────────────────┐          │

│  ┌──────────────────────────┐          │

│  └──────────────────────────┘          │
└─────────────────────────────────────────┘
```

### 新增文件清单

| 文件路径 | 类型 | 说明 |
|---------|------|------|
| `jellystudy-api/.../EvaluationService.java` | 接口 | 新增chatWithAI和generateKnowledgeGraph方法 |
| `jellystudy-evaluation-service/.../EvaluationServiceImpl.java` | 实现 | AI对话和图谱生成核心逻辑 |
| `jellystudy-web/.../AssistantController.java` | 控制器 | 处理前端请求的REST API |
| `jellystudy-web/.../fragments/assistant-widget.html` | 组件 | 悬浮窗UI + JavaScript |
| `jellystudy-web/.../templates/layout.html` | 模板 | 全局引入智能助手组件 |
| `jellystudy-web/.../templates/question/detail.html` | 模板 | 问题详情页添加标记按钮 |

### 数据存储策略

**Session存储（临时数据）：**
- `conversationHistory`: 对话历史记录（最多20条）
- `interestedKnowledgePoints`: 兴趣知识点列表

**特点：**
- ✅ 无需数据库改造
- ✅ 用户关闭浏览器后自动清除
- ✅ 轻量级实现，适合当前规模

## 🐛 故障排查

### 常见问题

**1. 悬浮窗不显示**
```
检查项:
- 浏览器控制台是否有JavaScript错误
- 是否正确执行了 docker compose up --build
- 清除浏览器缓存后刷新页面
```

**2. AI对话无响应**
```
检查项:
- evaluation-service容器是否正常运行: docker logs jellystudy-evaluation-service
- DeepSeek API Key是否有效
- 网络连接是否正常（需要访问外网）
```

**3. 知识图谱生成失败**
```
检查项:
- 是否至少标记了1个知识点
- AI返回的数据格式是否正确
- ECharts是否成功加载（F12查看Network）
```

**4. 标记按钮无效**
```
检查项:
- 当前页面是否为问题详情页（URL包含/question/{id}）
- 浏览器控制台是否报错: markInterest is not defined
- 刷新页面后重试
```

### 日志查看

```bash
# 查看evaluation-service日志
docker logs jellystudy-evaluation-service --tail 100 -f

# 查看web服务日志
docker logs jellystudy-web --tail 100 -f
```

## 📝 使用建议

### 最佳实践

1. **标记知识点时**
   - 选择真正感兴趣或需要深入理解的主题
   - 一次不要超过10个（图谱会更清晰）
   - 定期清理不再感兴趣的点

2. **与AI对话时**
   - 问题尽量具体明确
   - 可以追问细节（利用上下文记忆）
   - 复杂问题可以分步提问

3. **查看知识图谱时**
   - 拖拽节点调整布局
   - 悬停查看详细描述
   - 利用关系类型理解学习顺序
