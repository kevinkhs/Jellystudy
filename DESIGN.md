# JellyStudy 设计规范 (Design Specification)

> 版本: 1.0 | 最后更新: 2026-06-03 | 维护者: kevin

---

## 1. 产品概述

**JellyStudy（博识尊）** 是一款 AI 驱动的智能学习平台，面向大学生和终身学习者，提供题目练习、AI 智能问答、知识图谱可视化和兴趣追踪功能。

### 1.1 目标用户

| 用户类型 | 特征 | 核心需求 |
|---------|------|---------|
| 大学生 | 18-25岁，课业压力大，习惯移动端+PC双端使用 | 快速找到答案、高效复习、知识点关联理解 |
| 终身学习者 | 25-40岁，利用碎片时间学习，注重系统性 | 结构化知识体系、AI辅助答疑、个性化推荐 |

### 1.2 产品定位

> **"让知识像果冻一样清晰透明"** — JellyStudy 不是题库，而是你的 AI 学习伙伴。

核心差异化：
- **AI 助手悬浮窗**：全局可用的智能对话 + 知识图谱
- **兴趣标记系统**：标记感兴趣的知识点，自动生成个性化学习路径
- **知识图谱可视化**：将零散知识点串联成网络

## 2. 设计原则

### 2.1 核心价值观

| 原则 | 描述 |
|------|------|
| **清晰 Clarity** | 信息层级分明，用户3秒内找到目标内容 |
| **温暖 Warmth** | 学习不应冰冷，用柔和的视觉语言降低焦虑感 |
| **智能 Intelligence** | AI能力自然融入界面，不喧宾夺主 |
| **专注 Focus** | 减少干扰元素，让注意力留在内容本身 |

### 2.2 设计哲学

```
┌─────────────────────────────────────────────┐
│           内容为王 (Content First)            │
│                                             │
│   功能 > 装饰    可读性 > 创意    简洁 > 繁复   │
│                                             │
│   每一个像素都应该有存在的理由                   │
└─────────────────────────────────────────────┘
```

## 4. 排版规范 (Typography)

### 4.1 字体栈

```css
font-family: -apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto,
             'Helvetica Neue', 'PingFang SC', 'Microsoft YaHei', sans-serif;
```

- **优先使用系统字体**：保证跨平台一致性和加载速度
- 中文回退：`PingFang SC`(macOS) → `Microsoft YaHei`(Windows)
- 不引入外部字体文件，保持轻量

### 4.2 字号层级

| 层级 | 字号 | 字重 | 行高 | 使用场景 |
|------|------|------|------|---------|
| H1 - 页面标题 | 28px | 700 | 1.3 | Header 中的产品名 "JellyStudy" |
| H2 - 区块标题 | 22px | 600 | 1.4 | 卡片标题（最新问题、热门问题等） |
| H3 - 子标题 | 18px | 600 | 1.5 | 问题详情页标题、知识点名称 |
| Body - 正文 | 16px | 400 | 1.5~1.6 | 问题列表项、回答内容、描述文本 |
| Small - 辅助 | 14px | 400 | 1.5 | 表单标签、按钮文字 |
| Tiny - 元数据 | 13px | 400 | 1.4 | 时间戳、浏览量、作者名、标签文字 |

### 4.3 字重规范

| 字重 | 值 | 使用场景 |
|------|-----|---------|
| Regular | 400 | 正文、说明文字 |
| Medium | 600 | 标题、按钮文字、问题标题 |
| Bold | 700 | H1页面标题、统计数据数字(32px) |

## 5. 间距与布局 (Spacing & Layout)

### 5.1 间距系统 (8px Base Grid)

基于 8px 网格系统：

| Token | 值 | 使用场景 |
|-------|-----|---------|
| xs | 4px | 标签内边距、紧凑间距 |
| sm | 8px | 行内元素间距、小间隙 |
| md | 12px~15px | 表单组间距、列表项内间距 |
| lg | 20px~25px | 卡片内边距、区块间距 |
| xl | 30px | 内容区到Header的顶部距离 |
| xxl | 40px+ | 页面大区块间隔 |

### 5.2 容器规范

```css
.container {
    max-width: 1200px;     /* 最大内容宽度 */
    margin: 0 auto;        /* 居中 */
    padding: 0 20px;       /* 左右留白 */
}
```

### 5.3 响应式断点

| 断点 | 宽度 | 布局变化 |
|------|------|---------|
| Mobile | < 768px | 单列布局，导航折叠为汉堡菜单 |
| Tablet | 768px - 1024px | 双列布局（2fr 1fr），侧栏收窄 |
| Desktop | > 1024px | 标准三栏/两栏布局 |

当前首页采用 **Grid 两栏布局**：`grid-template-columns: 2fr 1fr`（主内容区 : 侧边栏）

## 7. 交互动效 (Interaction & Motion)

### 7.1 全局过渡规则

所有交互元素统一使用 `transition`：

| 属性 | 时长 | 缓动函数 |
|------|------|----------|
| 颜色变化 | 0.3s | ease |
| 背景变化 | 0.3s | ease |
| 阴影变化 | 0.3s | ease |
| 透明度 | 0.3s | ease |
| Transform | 0.3s | ease |

### 7.2 已定义动效

| 动效名称 | 应用对象 | 参数 |
|----------|---------|------|
| **bounce** | AI助手图标 | 2s infinite, translateY(0 ↔ -5px) |
| **hover-scale** | 按钮/卡片 | scale(1.05) 或 scale(1.1) |
| **hover-bg-slide** | 列表项 | 背景色滑动 + 圆角展开 |
| **focus-glow** | 输入框聚焦 | border变色 + 外发光环 |

### 7.3 动效原则

- **不要为了动效而动效**：每个动画必须有功能性目的
- **时长 ≤ 300ms**：超过这个时间会让用户感觉拖沓
- **尊重减少动画偏好**：未来应支持 `prefers-reduced-motion`

## 8. 页面结构 (Page Architecture)

### 8.1 页面清单

| 页面 | 路径 | 核心组件 |

## 9. 技术约束 (Technical Constraints)

### 9.1 技术栈

| 层 | 技术 |
|----|------|
| 后端框架 | Spring Boot 2.7.18 |
| 模板引擎 | Thymeleaf (HTML5 mode) |
| 前端样式 | 内联 `<style>`（Thymeleaf fragment 会过滤独立 style 标签） |
| 前端脚本 | 外部 JS 文件 (`/js/assistant.js`) |
| 图表库 | ECharts 5.4.3 (CDN) |
| 部署方式 | Docker Compose 多容器编排 |

### 9.2 重要注意事项

> **⚠️ Thymeleaf Fragment 样式限制**
>
> 由于项目使用 `th:replace` 引入组件片段，独立的 `<style>` 标签中的 CSS 可能被 Thymeleaf 引擎过滤掉。
> **解决方案**：关键样式必须写为 **内联 style 属性** 或确保样式定义在使用该 fragment 的父模板中。
>
> 当前已知受影响的组件：
> - AI 助手悬浮窗按钮尺寸和图标样式
> - 未来新增的 fragment 组件均需注意此问题

### 9.3 浏览器兼容性

| 浏览器 | 最低版本 |
|--------|---------|
| Chrome | 90+ |
| Firefox | 88+ |
| Safari | 14+ |
| Edge | 90+ |

不支持 IE。CSS 使用 Grid 和 Flexbox 作为主要布局方案。

## 10. 改进方向 (Future Improvements)

以下为期望达到的设计目标（按优先级排序）：

### P0 - 必须改进

- [ ] **提取公共CSS为独立文件**：解决 Thymeleaf style 过滤问题后，将重复的样式代码抽取到 `static/css/common.css`
- [ ] **响应式适配**：当前未做移动端适配，需添加 Mobile 断点的汉堡菜单和单列布局
- [ ] **暗色模式**：添加 Dark Mode 支持，通过 CSS 变量实现主题切换

### P1 - 应该改进

- [ ] **统一设计Token**：将硬编码的颜色/字号/间距全部改为 CSS 自定义属性（CSS Variables）
- [ ] **骨架屏加载**：列表/详情页加载时显示 Skeleton Screen，替代空白等待
- [ ] **空状态设计**：无数据时展示友好的插画+引导文案（当前仅有纯文字提示）
- [ ] **微交互增强**：按钮点击反馈（ripple效果）、点赞心跳动画、页面切换过渡

### P2 - 可以改进

- [ ] **字体优化**：考虑引入 Inter 或 Noto Sans SC 作为增强字体
- [ ] **图标系统**：从 emoji 文字图标迁移到 SVG Icon 库（如 Lucide Icons 或 Phosphor Icons）
- [ ] **动画库**：引入 Animate.css 或 Framer Motion 处理复杂入场动画
- [ ] **品牌插图**：为空状态、错误页、Loading 状态设计专属 SVG 插画
